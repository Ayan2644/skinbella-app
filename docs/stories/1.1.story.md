## Story

**As a** usuária do Skinbella App,
**I want** um quiz com perguntas estratégicas que se adaptam dinamicamente baseadas nas minhas respostas anteriores,
**so that** meu diagnóstico capte informações realmente relevantes ao meu problema específico de pele e resulte em uma análise mais precisa e personalizada.

## Acceptance Criteria

1. A interface `QuizQuestion` em `src/lib/quizData.ts` é estendida com campo opcional `branching?: BranchRule[]` — sem quebrar perguntas existentes (backward compatible)
2. O tipo `BranchRule` é definido como: `{ when: { questionId: string; includes: string | string[] }; insertAfter: string; question: QuizQuestion }` — totalmente type-safe (sem `any`)
3. A função pura `getVisibleQuestions(baseQuestions: QuizQuestion[], answers: Record<string, string | string[]>): QuizQuestion[]` é criada e exportada de `src/lib/quizData.ts`
4. Pelo menos **2 regras de branching** são implementadas:
   - Se `incomodos` inclui `'acne'` → inserir pergunta `acne_tipo` após `incomodos`
   - Se `tipo_pele` é `'sensivel'` → inserir pergunta `sensibilidade_gatilhos` após `tipo_pele`
5. Pelo menos **3 novas perguntas estratégicas** são adicionadas ao `quizQuestions`:
   - `ciclo_hormonal` — captura influência hormonal nas manchas/acne (nova dimensão estratégica)
   - `acne_tipo` — pergunta condicional (aparece só se acne selecionada em `incomodos`)
   - `sensibilidade_gatilhos` — pergunta condicional (aparece só se `tipo_pele === 'sensivel'`)
6. `Quiz.tsx` utiliza `getVisibleQuestions(baseQuestions, answers)` para computar `activeQuestions` dinamicamente a cada mudança de resposta
7. `QuizProgress` exibe progresso correto com o total dinâmico de perguntas visíveis (sem hard-code do total)
8. A navegação (next/prev) funciona corretamente — ao voltar de uma pergunta condicional que foi removida, o step é ajustado para não ultrapassar os limites da lista atual
9. `npm run typecheck` passa sem erros — zero uso de `any` em código novo
10. `npm run lint` passa sem erros
11. Testes unitários para `getVisibleQuestions()` cobrem:
    - Fluxo padrão (sem branching ativado)
    - Fluxo com `incomodos` incluindo `'acne'` → pergunta condicional aparece
    - Fluxo com `tipo_pele === 'sensivel'` → pergunta condicional aparece
    - Combinação de ambos os branchings simultâneos

### Edge Case: Respostas Órfãs

Quando o usuário desmarca `acne` em `incomodos`, a pergunta `acne_tipo` some da lista mas a resposta `answers.acne_tipo` continua em memória. **Comportamento esperado: ignorar silenciosamente** — a `getVisibleQuestions()` não retorna a pergunta, então o skinEngine (Story 1.2) simplesmente não terá `acne_tipo` no perfil, o que é correto. **Não limpar** as respostas órfãs em Quiz.tsx — isso quebraria a navegação prev/next.

### Design da Solução — BranchRule

```typescript
// Em src/lib/quizData.ts (novo código)
export interface BranchRule {
  when: {
    questionId: string;               // ID da pergunta gatilho
    includes: string | string[];       // Valor(es) que ativam o branch
  };
  insertAfter: string;                // ID da pergunta após a qual a nova pergunta é inserida
  question: QuizQuestion;            // Pergunta condicional a inserir
}

// Extensão de QuizQuestion (adicionar ao existente)
export interface QuizQuestion {
  // ... campos existentes mantidos ...
  branching?: BranchRule[];          // Regras de branching opcionais
}
```

### Design da Solução — `getVisibleQuestions()`

```typescript
// Algoritmo de referência (não copiar literalmente — implementar type-safe)
export function getVisibleQuestions(
  baseQuestions: QuizQuestion[],
  answers: Record<string, string | string[]>
): QuizQuestion[] {
  const result: QuizQuestion[] = [];
  const inserted = new Set<string>(); // evita duplicatas de perguntas condicionais

  for (const q of baseQuestions) {
    result.push(q);

    // Verificar se alguma regra de branching desta pergunta é ativada
    if (q.branching) {
      for (const rule of q.branching) {
        const answer = answers[rule.when.questionId];
        const trigger = Array.isArray(rule.when.includes) ? rule.when.includes : [rule.when.includes];
        const isActivated = Array.isArray(answer)
          ? trigger.some(t => (answer as string[]).includes(t))
          : trigger.includes(answer as string);

        if (isActivated && !inserted.has(rule.question.id)) {
          result.push(rule.question);
          inserted.add(rule.question.id);
        }
      }
    }
  }

  return result;
}
```

**Nota:** O campo `branching` é declarado na pergunta que antecede a posição onde a condicional deve aparecer — não na pergunta condicional em si.

### Atualização em `Quiz.tsx`

```typescript
// Padrão de uso (referência de implementação)
const baseQuestions = quizStorage.getActiveQuestions();

// activeQuestions derivado de answers — recomputa a cada render
const activeQuestions = getVisibleQuestions(baseQuestions, answers);
const currentQ = activeQuestions[step];
const totalSteps = activeQuestions.length;
```

**Atenção em `next()`**: após `setAnswers`, o próximo render já recomputa `activeQuestions`. Verificar `step < activeQuestions.length - 1` com a lista NOVA é crítico para evitar ir além do fim quando uma pergunta condicional é removida.

Padrão recomendado:
```typescript
const next = () => {
  const updatedAnswers = { ...answers, [currentQ.id]: answers[currentQ.id] };
  const updatedQuestions = getVisibleQuestions(baseQuestions, updatedAnswers);
  if (step < updatedQuestions.length - 1) {
    setStep(s => s + 1);
  } else {
    storage.saveAnswers(updatedAnswers);
    setPhase('processing');
  }
};
```

### Perguntas Estratégicas — Spec Completa

**`ciclo_hormonal`** (pergunta padrão — sempre visível, após `incomodos`):
- Rationale: 70% das mulheres têm alterações de pele ligadas ao ciclo — dado crítico ausente hoje
- Localização no array: inserir antes de `selfie`, após `incomodos`
- Opções sugeridas: `nao_se_aplica` ("Não se aplica"), `regular` ("Regular, sem grandes variações"), `irregular` ("Irregular, com alterações na pele"), `uso_hormonal` ("Uso anticoncepcional ou TRH")

**`acne_tipo`** (condicional — `incomodos.includes('acne')`):
- Rationale: acne comedonal vs inflamatória vs hormonal → tratamentos COMPLETAMENTE diferentes no skinEngine v2
- Opções: `hormonal` ("Hormonal - piora no TPM"), `comedonal` ("Cravos e poros obstruídos"), `inflamatorio` ("Espinhas e inflamação"), `misto` ("Combinação de tipos")

**`sensibilidade_gatilhos`** (condicional — `tipo_pele === 'sensivel'`):
- Rationale: sensibilidade a sol vs cosméticos vs alimentos → orienta quais ingredientes recomendar/evitar
- Tipo: multi-chips (multiSelect: true)
- Opções: `sol` ("Sol e calor"), `cosmeticos` ("Cosméticos e perfumes"), `alimentos` ("Alimentos específicos"), `stress` ("Estresse"), `hormonal` ("Hormônios")
