# cientista-quality-gate

Quality gate para outputs do agente **Dra. Lara (cientista)**.
Executar antes de marcar qualquer documento como ✅ aprovado.

---

## Checklist — Claims Científicos

### 1. Fundamentação
- [ ] Cada claim de efeito tem mecanismo documentado
- [ ] Mecanismos são fisiologicamente plausíveis
- [ ] Referência ao estudo ou mecanismo está implícita ou explícita

### 2. Linguagem
- [ ] Nenhum uso de "elimina", "cura", "definitivamente", "garantido"
- [ ] Claims hedgeados onde evidência é moderada: "contribui para", "apoia"
- [ ] Linguagem acessível mas não superficial

### 3. Prazos
- [ ] Prazos são conservadores (preferir D60 a D30 se incerto)
- [ ] D7 é possível como resultado interno mesmo sem visível
- [ ] Nenhum prazo absoluto sem hedge de variabilidade individual

### 4. Regulatório
- [ ] Nenhum claim de tratamento médico (ANVISA)
- [ ] Sem afirmações de cura de doença
- [ ] Distinção clara entre saúde e estética

### 5. Diferenciação Caps × Serum
- [ ] Caps sempre descrito como sistêmico/oral
- [ ] Serum sempre descrito como tópico/local
- [ ] Sinergia entre os dois documentada quando relevante

### 6. Formato
- [ ] Output é documento .md (nunca código)
- [ ] Estrutura por perfil presente (8 perfis)
- [ ] Fonte das informações rastreável
